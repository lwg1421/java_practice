# 연산자
## 연산자의 종류
- 자바의 연산자는 연산 결과의 특징에 따라 세가지로 나눌 수 있다.
  - 연산 결과가 특정 값이 나오는 산술, 증감, 비트, 시프트 연산자
  - 연산 결과가 참(true)과 거짓(false)중 하나의 값으로만 나타나는 비교, 논리 연산자
  - 값을 반환하는 것이 나닌 값을 대입하는 의미를 지니고 있는 대입 연산자와 삼항 연산자
## 연산자의 연산 방법
### 산술 연산자와 증감 연산자
- 산술 연산자(+, -, *, /, %)
  - 가장 기본적인 연산자
  - 사칙연산과 나머지 연산이 있다.
  - '정수/정수' 형태의 나누기를 할 때 주의 해야할 점이 있다.
    - int/int = int이므로 5/2의 값은 2/5가 아니라 2이다.
  - 나머지 연산(%)는 말 그대로 나누기를 수행한 이후의 나머지를 나타내는데 이를 '모듈로 연산' 이라고 한다.
- 증감 연산자(++, --)
  - 변수값을 1씩 증가 또는 감소시킨다    .
  - 전위형(++변수)
    - 변수 앞에 붙는다.
    - 다른 모든 명령보다 먼저 실행된다.
    - 우선순위가 일등 
  - 후위형(변수++)   
    - 변수 뒤에 붙는다.
    - 가장 나중에 실행된다.
    - 우선순위가 꼴등 
### 비트 연산자
>> 말그대로 비트 단위의 연산자이다.
>> AND(&), OR(|), XOR(^), NOT(!)이 있다.
- AND(&)
  - 두 값이 모두 1일때만 1인 연산자 
- OR(|)
  - 두 값이 모두 0일 때만 0인 연산자
  - 둘 중 하나라도 1이면 1이라는 의미 
- XOR(^)
  - 두 값이 다르면 0, 같으면 1인 연산자 
- NOT(!)
  - 0은 1로, 1은 0으로 반전시키는 연산자
### 시프트 연산자
>> 비트의 위치를 좌우로 이동하는 연산자로 산술 시프트(<<, >>)와 논리 시프트(>>>)가 있다.
- 산술 시프트
  - 숫자의 부호 비트는 유지하면서 나머지 비트를 왼쪽(<<) 또는 오른쪽(>>)으로 이동하는 연산자. 
- 논리 시프트
  - 부호 비트를 포함해 전체 비트를 오른쪽으로 이동시키는 연산 
### 비교 연산자
>> 비교 연산자는 크게 크기 비교와 등가 비교로 나눌 수 있다.<br>
>> 연산 결과는 불리언 자료형, 즉 true나 false중 하나를 나타낸다.
- 비교 연산자에서 중요한 것은 등가 비교(==, !=)를 할 때 비교 대상이 무엇이냐는 것이다.
  - 스택 메모리 값을 비교한다.  
  - 스택 메모리의 값은 기본 자료형일 경우와 참조 자료형일 경우가 서로 다르므로 이에 주의할 필요가 있다
    - ex) RelationOperator파일의 String의 등가 비교 결과 참조!! 
### 논리 연산자
>> 논리 연산자는 비트 연산자와 매우 비슷하지만, 피연산자로 불리언 값만 올 수 있고, 연산 결과 또한 불리언 타입만을 지닌다.
- 논리 AND(&&)
  - 두 값이 모두 true일 때만 true
  - 나머지는 모두 false값을 가짐 
- 논리 OR(||)
  - 두 값이 모두 false일 때만 false
  - 나머지는 모두 true 
- 논리 XOR(^)
  - 두 값이 다를 때는 true
  - 두 값이 같을 때는 false 
- 논리 NOT
  - true와 false를 반전시키는 연산자
>> <b>쇼트 서킷</b><br>
>> 쇼트 서킷은 연산을 수행하는 과정에서 결과가 이미 확정됐을 때 나머지 연산 과정을 생략하는 것을 말한다.<br>
>> 예를 들어 (5>3) || (3<2)을 수행하고자 할 때 왼쪽의 결과가 true이므로 오른쪽 결과와 상관없이 결관 true이다.<br>
>> 따라서 이때는 오른쪽 항인 (3<2)를 아예 읽지도 않는 것이 '쇼트 서킷' 이다.<br>
>> <b>논리 연산자로 연산을 수행할 때에는 쇼트 서킷이 적용되지만<br>
>> 비트 연산자로 논리 연산을 수행하면 쇼트 서킷이 적용되지 않는다.</b>
### 대입 연산자
- 대입 연산자(=)는 오른쪽 피연산자의 연산 결과를 왼쪽 변수에 대입하는 연산자
- 대입 연산자는 다른 연산자와 결합해 축약된 형태로 사용도될 수 있다.
  - 축약 예시
  ```java
  a+=b  // a = a+b 의 의미
  a-=b  // a = a-b 의 의미
  ```
 - 축약 표현은 단순히 수식만 간략화한 것이므로 실행 결과에는 아무런 영향을 미치지 않는다.
### 삼항 연산자
>> 삼항 연산자는 다음과 같이 자바 연산자 중 유일하게 세개의 피연산자가 있는 연산자이다.
- 삼항 연산자의 구성
```
(참 또는 거짓) ? 참일 때 연산 결과 : 거짓일 때 연산 결과
```
- 물음표 앞에는 불리언 자료형인 참 또는 거짓만 올 수 있다.
- 불리언 자료형(true or false)이 직접올 수 있지만, 주로 연산 결과가 불리언 자료형인 논리 연산자와 비교 연산자가 위치한다.
- 이 값이 참일 경우 콜론(:)의 앞쪽, 거짓일 경우에는 뒤쪽이 연산 결과가 된다.
